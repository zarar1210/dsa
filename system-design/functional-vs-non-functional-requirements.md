## **Functional and Non-Functional Requirements**

### _Functional Requirements:_

- It defines **what** the system should perform.
- Functional requirements specify the exact features and actions the system must execute to meet business needs and user expectations.
- They describe what the system should do and how it should interact with users and other systems.
- The focus is on system behavior and functionality.
- Common examples include payment processing, data handling, search, authentication, and report generation.
- They focus on business logic and user interactions and are verified through functional testing (unit testing, integration testing, etc.).

## **Non-Functional Requirements**

- They define **how** a system should perform.
- Focus areas include:
  - Performance
  - Security
  - Usability
  - Reliability
  - Scalability
  - Maintainability
  - Portability

### **_Example: Online Banking System_**

**Functional Requirements:**

1. Users should be able to log in with their username and password.
2. Users should be able to check their account balance.
3. Users should receive notifications after making a transaction.

**Non-Functional Requirements:**

1. The system should respond to user actions in less than 2 seconds.
2. All transactions must be encrypted and comply with industry security standards.
3. The system should handle up to 100 million users with minimal downtime.
